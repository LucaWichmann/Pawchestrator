# ADR 0011 — Attribution footer on Pawchestrator-generated GitHub artifacts

**Status:** Accepted  
**Date:** 2026-05-26

## Context

Pawchestrator produces GitHub artifacts (PR bodies, Grill criteria sections) that readers encounter without navigating to the run comment. There is no signal at the artifact itself that it was AI-generated.

## Decision

PR bodies (regular and epic) and the `## Pawchestrator Suggested Criteria` section appended by Grill each carry a one-line attribution footer:

```markdown
---
<sub>Generated by [Pawchestrator](https://github.com/LucaWichmann/Pawchestrator)</sub>
```

`<sub>` is the only GitHub Markdown mechanism for visibly reduced text; CSS color/opacity is unavailable. A horizontal rule creates visual separation without being loud.

Three alternatives were rejected:

- **Run-state comment**: already carries run ID, stage labels, and timestamps that make the automated origin obvious. Adding attribution would be redundant.
- **HTML comment `<!-- … -->`**: invisible to readers; defeats the purpose.
- **Emoji prefix (🐾)**: competes with label copy already using the paw motif elsewhere.

The Grill footer is placed *inside* the `## Pawchestrator Suggested Criteria` section (last content before the next heading boundary). This means re-grill replaces the entire section, including the footer, so deduplication is automatic and no extra tracking is required. Placing the footer *after* the section would cause it to stack on each re-grill round.

Attribution is template-driven (not LLM-generated), so it does not conflict with ADR 0002's prohibition on LLM text in GitHub artifacts.
