# ADR 0011 - Attribution footer on Pawchestrator-generated GitHub artifacts

**Status:** Accepted
**Date:** 2026-05-26

## Context

Pawchestrator produces GitHub-visible artifacts that readers can encounter without
navigating to local run state. These include PR bodies, review comments, run
comments, Grill comments, Grill criteria sections, and follow-up issues created
from review suggestions. Each artifact should visibly identify that Pawchestrator
generated it.

## Decision

Pawchestrator-generated GitHub-visible text carries a one-line attribution footer:

```markdown
---
<sub>Generated by [Pawchestrator](https://github.com/LucaWichmann/Pawchestrator)</sub>
```

`<sub>` is the only GitHub Markdown mechanism for visibly reduced text; CSS
color/opacity is unavailable. A horizontal rule creates visual separation without
being loud.

The footer is appended at artifact generation sites, not in the low-level GitHub
API client. This keeps ordinary issue-body patches, such as checkbox updates,
from stamping user-authored text.

The helper is idempotent: it removes an existing matching footer before appending
the footer again, so edited comments and regenerated bodies do not accumulate
duplicate footers.

The Grill footer is placed inside the `## Pawchestrator Suggested Criteria`
section, as the last content before the next heading boundary. New criteria are
inserted above the footer, and re-grill updates the section without stacking
footers.

Attribution is template-driven, not LLM-generated, so it does not conflict with
ADR 0002's prohibition on LLM text in GitHub artifacts.
